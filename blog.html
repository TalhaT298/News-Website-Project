var: 1. The scope of a var variable is functional scope. 2. It can be updated and re-declared into the scope. 3. It can be declared without initialization. 4. It can be accessed without initialization as its default value is "undefined". Let: 1. The scope of a let variable is block scope. 2. It can be declared without initialization. 3. It can be updated but cannot be re-declared into the scope. 4. It cannot be accessed without initialization otherwise it will give 'referenceError'. Const: 1. The scope of a const variable is block scope. 2. It cannot be updated or re-declared into the scope. 3. It cannot be declared without initialization.

Regular Functions : 1. Inside of a regular JavaScript function, this value (aka the execution context) is dynamic. 2. The regular functions are the usual way to define methods on classes. 3. As seen in the previous section, the regular function can easily construct objects. 4. Return expression statement returns the result from a function. Arrow Functions : 1. The behavior of this inside of an arrow function differs considerably from the regular function's this behavior. The arrow function doesn't define its own execution context. 2. A consequence of this resolved lexically is that an arrow function cannot be used as a constructor. 3. On the other side, no arguments special keyword is defined inside an arrow function. 4. You can return values from the arrow function the same way as from a regular function, but with one useful exception.
